# ADR-001 Encryption Scope

## Context

Deciding how much data to encrypt and on what point in order to balance privacy and usability.

## Decision

 - Cost names - client side encryption. There is absolutely no reason backend needs to have this information given our privacy focus!
 - Cost amounts - server side at rest encryption. Privacy tradeoff to give backend anything to do. Without this information backend is basically data storage, which limits learning experience of this exercise.
 - Group members - not encrypted. It might be possible to do back and forth between frontend and backend to accommodate this, but it's definitely beyond the scope of MVP 

## Consequences

The only actually encrypted part from client side perspective is what the actual cost is for, which is not as private as I'd like, but for the sake of learning it will have to do

## Future considerations

During hardening phase - consider pushing bulk of logic into frontend, so that we can hide more data from backend
